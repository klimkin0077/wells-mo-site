# Проверка исходного HTML WELLS-MO

Дата проверки: 2026-05-18

Проверка выполнена по исходному ответу сервера **без выполнения JavaScript**. Для этого были получены HTML-ответы главной страницы и внутреннего маршрута `/remont-kolodcev` через `curl`.

## Ключевой вывод

Сайт в текущем виде **не является SEO-готовым для публикации**, если требование состоит в том, чтобы **H1, услуги, цены, города и FAQ находились прямо в HTML-ответе сервера**. Сейчас в исходном HTML есть только базовый шаблон SPA с контейнером `#root`, а основной контент дорисовывается уже клиентским JavaScript.

## Что найдено в реальном HTML

| Проверяемый блок | Главная `/` | Маршрут `/remont-kolodcev` | Вывод |
|---|---:|---:|---|
| **H1** | Нет | Нет | В исходном HTML тега `<h1>` не найдено |
| **Блок услуг** | Нет | Нет | Названия услуг не присутствуют как видимый HTML-контент страницы |
| **Блок цен** | Нет | Нет | Цены в HTML-ответе отсутствуют |
| **Города / география** | Нет как контент | Нет как контент | Города встречаются только в `meta keywords`, но не в теле страницы |

| **SPA-контейнер** | Да | Да | Есть только `<div id="root"></div>` |

## Что это означает practically

Для пользователя страница визуально выглядит наполненной, потому что браузер выполняет React-код и вставляет контент в DOM после загрузки. Но если смотреть именно **первичный HTML-ответ**, то обязательные коммерческие SEO-блоки там отсутствуют. Это именно тот риск, о котором вы предупредили.

## Решение по публикации

На текущем этапе рекомендация однозначная: **не публиковать и не заливать** эту версию как «SEO-готовую».

## Что нужно сделать дальше

Есть три реалистичных пути исправления в режиме предпросмотра.

| Вариант | Суть | Плюсы | Минусы |
|---|---|---|---|
| **A. Быстрый компромисс** | Добавить в исходный HTML статический SEO-фолбэк с H1, услугами, ценами, городами и FAQ | Быстро, можно проверить через `curl` сразу | Архитектурно грязнее, нужно следить за дублированием контента |
| **B. Правильный путь** | Настроить prerender/SSG для ключевых маршрутов, чтобы каждая страница отдавалась уже с готовым HTML | Лучший вариант для SEO и масштабирования | Потребует переделки сборки и маршрутизации |
| **C. Минимальный вариант** | Добавить большой `<noscript>`/SEO-блок в `index.html` | Очень быстро | Самый слабый вариант, годится только как временная мера |

Если хотите, следующим шагом я могу **сделать вариант B или A только в preview**, после чего снова показать вам `curl` и уже доказать, что **H1, услуги, цены, города и FAQ реально появились в HTML**.
