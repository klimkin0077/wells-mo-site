# WELLS-MO

**WELLS-MO** — многостраничный сайт услуг по чистке, ремонту, углублению колодцев и подводке воды в дом по Московской области.

## Что находится в проекте

Проект собран на **React 19**, **Vite** и **Tailwind CSS 4**. Основные пользовательские страницы и контентная логика находятся во фронтенде, а небольшой сервер нужен для отдачи production-сборки и корректной работы клиентской маршрутизации.

| Раздел | Назначение |
| --- | --- |
| `client/src/pages/SitePages.tsx` | Основные страницы сайта и визуальные секции |
| `client/src/lib/site-content.ts` | Контентная модель: услуги, цены, FAQ, отзывы, локации |
| `client/src/index.css` | Глобальная тема, адаптивность и визуальные токены |
| `server/index.ts` | Production-сервер для раздачи сборки |

## Запуск локально

Сначала установите зависимости, затем запустите проект в режиме разработки.

```bash
pnpm install
pnpm dev
```

По умолчанию сайт поднимется на локальном порту разработки Vite.

## Проверка и production-сборка

Перед публикацией в репозиторий и перед деплоем рекомендуется проверить типы и собрать production-версию.

```bash
pnpm check
pnpm build
```

Если сборка прошла без ошибок, production-версию можно запустить так:

```bash
pnpm start
```

## Публикация из GitHub

Для развёртывания на сервере нужен Node.js 22+ и `pnpm`.

| Шаг | Команда |
| --- | --- |
| Клонировать репозиторий | `git clone <repo_url>` |
| Перейти в проект | `cd premium-wells-site` |
| Установить зависимости | `pnpm install` |
| Собрать проект | `pnpm build` |
| Запустить production | `pnpm start` |

Если сайт запускается за reverse proxy, достаточно проксировать входящий трафик на порт `3000` или на значение `PORT`, переданное через переменную окружения.

## Что важно не трогать без необходимости

Не стоит хаотично менять структуру `SitePages.tsx`, контентную модель в `site-content.ts` и глобальные токены в `index.css` без проверки мобильной версии и production-сборки. Эти файлы задают общую логику маршрутов, текста и адаптивного поведения.

## Рекомендуемый порядок дальнейших обновлений

Сначала обновляйте тексты и контентную модель, затем проверяйте мобильную версию, после этого прогоняйте `pnpm check` и `pnpm build`, и только потом отправляйте изменения в основную ветку репозитория.
