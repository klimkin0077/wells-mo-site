# План точечного усиления wells-mo.ru без редизайна

## Главные ограничения

Работа ведётся только в режиме предпросмотра. Изменения не отправляются в GitHub и не публикуются до отдельной команды пользователя `одобряю, заливай`.

Визуальная база сайта сохраняется: тёмный фон, золотые акценты, крупные заголовки, карточки услуг, дорогая инженерная подача и текущая мобильная логика. Любые улучшения должны быть эволюционными, а не радикальными.

## Что усиливаем поверх текущей структуры

| Направление | Что меняем | Как сохраняем стиль |
|---|---|---|
| Контакты и CTA | Убираем WhatsApp из основных каналов, добавляем Telegram и MAX, усиливаем desktop/mobile sticky CTA | Используем текущие тёмные панели, золотые акценты, скругления и glassmorphism |
| Форма «Задать задачу» | Расширяем поля, добавляем сбор текста заявки, действия mailto / Telegram / MAX / копирование | Оставляем текущую модальную логику и визуальный язык существующего диалога |
| SEO-структура | Дорабатываем страницы услуг, хлебные крошки, canonical, schema, robots, sitemap, 404, alias-маршруты | Не меняем основу навигации, а расширяем её точечно |
| Контент | Переводим тексты в более жёсткий, фактический, инженерный тон | Сохраняем текущую премиальную подачу, но снижаем избыточную «глянцевость» |
| Города | Усиливаем денежные направления и локальные акценты без тупого копипаста | Используем уже существующую локальную архитектуру и дополняем её приоритетными кластерами |
| Визуальная динамика | Добавляем лёгкие анимации, hover и микроэффекты | Только CSS/JS без тяжёлых библиотек и без поломки скорости |

## Приоритет реализации

Сначала нужно привести в порядок контактный слой: единый конфиг контактов, sticky CTA, обновлённая форма и действия для Telegram/MAX. Затем расширяется SEO-структура и маршруты услуг. После этого усиливаются денежные города, калькулятор и технические SEO-артефакты. Финальный этап — только браузерная проверка в предпросмотре, особенно на мобильной версии.

## Ключевые технические решения

Текущая модальная форма уже существует, поэтому её нужно не заменять, а расширять. Текущую нижнюю мобильную панель нужно переработать в четырёхкнопочный формат: звонок, Telegram, MAX, заявка. Для внутренних SEO-усилений следует опираться на уже существующие маршруты и контентную модель, а новые alias-пути и страницы встраивать поверх них без слома существующих URL.

## Что обязательно проверить в предпросмотре

| Зона проверки | Что должно работать |
|---|---|
| Desktop sticky CTA | Плавающая кнопка «Задать задачу» открывает форму без перекрытия контента |
| Mobile sticky bar | Кнопки Позвонить, Telegram, MAX и Заявка помещаются в экран и не перекрывают текст |
| Форма | Корректно формирует текст заявки, копирует его и открывает нужный канал связи |
| Страницы услуг | Имеют уникальные title/description/H1, FAQ, CTA, перелинковку и не выбиваются из дизайна |
| Денежные города | Отличаются по вводным акцентам и типовым задачам |
| Техническое SEO | Работают robots.txt, sitemap.xml, canonical, breadcrumb schema, FAQ schema, 404 и alias-страницы |
| Мобильная версия | Нет горизонтального скролла, hero не разваливается, поля формы удобны для пальца |
